This leaves a real gap:\n\n- OpenAlex ranks by citation count. A 2024 preprint on \"speculative decoding\" or \"sparse MoE routing\" will have <50 citations and will never surface.\n- The arXiv API's default `sortBy=relevance` uses Apache Lucene, which ranks by **query match**, not citation age. For cutting-edge/preprint goals this is strictly better.\n- Field prefix syntax (`ti:`, `abs:`, `cat:cs.LG`) allows targeted queries beyond keyword match.\n- Rate limit (1 req/3 sec) is fine for serial per-goal queries; the adapter only needs 1–2 calls per goal.\n- Abstracts are explicitly CC0 under arXiv's ToU — safe to store.\n\n**Adapter implementation notes:** The response is Atom XML. Parse with `xml.etree.ElementTree`. Map `<entry>/<summary>` → `abstract`, `<entry>/<id>` → `source_id` (strip `http://arxiv.org/abs/`), derive `arxiv_id` from same, `source_type=\"arxiv\"`. Authority score: use `<arxiv:comment>` or default to a fixed midpoint (e.g. 0.35) since citation counts are not in the arXiv API — you can enrich via Semantic Scholar if needed.\n\n**Recommendation:** Add as a parallel adapter alongside OpenAlex, weighted toward \"cutting-edge\" and \"systematic\" intents (`intent in {\"cutting-edge\", \"systematic\"}`). The existing `find_sources.py` loop over adapters accommodates this with one line.\n\n---\n\n## 3. Top-3 \"Do These First\"\n\n### (A) Semantic Scholar — biggest quality jump for lowest effort\n\n**Why first:** S2's paper bulk search endpoint (`/graph/v1/paper/search/bulk?query=...&fields=title,abstract,tldr,citationCount,externalIds,openAccessPdf`) returns ML-ranked results — it uses a trained ranker, not raw citation count. This directly fixes the tangential-paper problem. The TLDR field is a 20-word AI summary that can substitute as `abstract` for papers with no deposited abstract. The `externalIds` field includes `ArXiv`, `DOI`, `PubMed`, making cross-referencing free. It returns a Python-friendly JSON body that maps to `Source` in ~15 lines. A free API key takes 2 minutes to request and raises limits to 1 RPS dedicated. The biggest risk: the API License restricts commercial use without an Expanded License, but the project is academic/non-commercial, which is broadly permitted.\n\n**Impact:** Fixes the \"Raft returns wireless-blockchain\" problem immediately, since S2's ranker understands topic coherence better than citation-count-ranked keyword search.\n\n### (B) arXiv direct search — preprint gap for nearly zero extra work\n\n**Why second:** The codebase already knows how to handle `arxiv_id` sources (fulltext fetching, PDF URL construction). Promoting arXiv from fetch-only to search means one new adapter file of ~30 lines, mirroring `openalex.py` in structure. No key, no auth, no new dependencies (stdlib urllib + ElementTree). Gate it on `intent in {\"cutting-edge\", \"systematic\"}` so it doesn't flood beginner/crash-course goals with paper overload. This closes the \"recent ML paper has zero citations\" gap completely.\n\n### (C) Stack Exchange — fills the implementation gap with no paywall\n\n**Why third:** The \"build a working Raft implementation\" use case currently gets zero implementation guidance because all current adapters return theory papers. Stack Overflow Q&A is the de facto resource for implementation questions. Stack Exchange API v2.3 is well-documented, stable, and a free key (register at stackapps.com) gives 10,000 req/day. The `/search/advanced` endpoint supports `tagged=` filtering (e.g. `tagged=distributed-systems` or `tagged=consensus`) which can be derived from the existing `intent` classification. Content is CC BY-SA 4.0 — attribution is required (link back to question URL), which the `Source.url` field already provides. Gate this adapter on `intent=\"applied\"` or when the goal text contains implementation-signal words (build, implement, code, tutorial).\n\n---\n\n## 4. Things to Avoid or Flag\n\n- **SSRN:** No API. Economics preprints are better served by IDEAS/RePeC (if you can get approved access) or by OpenAlex filtering `primary_location.source.display_name` for economics journals.\n- **YouTube official API:** The `captions.download` endpoint costs 200 quota units and only works for manual captions. Auto-generated captions (the majority) are inaccessible. Use `youtube-transcript-api` (PyPI, jdepoix/youtube-transcript-api) which uses the undocumented `timedtext` endpoint and needs no key. Code results need heavy post-processing to produce a useful `abstract`. Defer indefinitely unless the specific use case is \"understand this repo.\"\n- **Crossref for primary search:** Crossref is an excellent enricher (DOI → metadata) but a poor primary search engine — it has no semantic ranking and returns broad results. Use it inside `fulltext.py` or a metadata enrichment pass, not as a DISCOVER adapter.\n- **PubMed over Europe PMC:** For LitNavigator's purpose, Europe PMC is strictly easier to use (no two-step ESearch+EFetch dance, 33M records, no key required, 10 req/sec) and covers PubMed content.
