# LitNavigator — Frontend: What Ships Today

**Branch:** `feat/open-world-digest` · **Updated:** 2026-06-21

The unified glass-box + user frontend was delivered in OW-6 (P6). This document describes every
implemented frontend feature, the UI architecture, and how it connects to the backend.

---

## One page, two views

The app serves a single session page at `/tutor/{sid}` that combines:

- **User view (Chat pane):** clean conversation — messages, quiz prompts, learner answers,
  artifact cards, and the recommend-next card at session end.
- **Glass-box pane:** full technical chain visible alongside the conversation — per-step
  skill/method/paper chips, live mastery and confidence scores, concept-map SVG, cited
  evidence per teaching turn, cost meter, and the recommend-next breakdown.

A **"Show research detail" toggle** (in the glass-box pane) switches between a lay summary
("what the tutor is doing and why") and the full technical chain (skill name, method, paper,
raw scores). This serves both a general learner and a technical evaluator on the same page.

## Routes

| Route | Template | Description |
|---|---|---|
| `GET /` | `index.html` | Redirects to most recent session (demo) or empty-state message |
| `GET /tutor` | `agent_home.html` | Landing page: enter any learning goal; lists available topics from the fixture |
| `GET /tutor/{sid}` | `agent.html` | **Main session page** — chat + glass-box, live SSE updates |
| `GET /sessions/{sid}/trace` | — | JSON trace endpoint (used by the glass-box SSE poller) |
| `GET /sessions/{sid}` | `index.html` | Legacy closed-world trace panel (still active for `demo-m2`/`demo-m3` runs) |

Server: `litnav/ui/server.py` (FastAPI). Launch: `python -m litnav.ui.server` → http://127.0.0.1:8000/tutor.

## Per-step skill / method / paper chips

Every agent step rendered in the glass-box carries three chips derived from
`litnav/ui/flow_meta.py::NODE_META`:

- **skill** — the high-level stage skill the node implements (e.g., `assess`, `teach`, `digest`)
- **method** — the research method or algorithm (e.g., "BKT / Rasch-IRT mastery update (never LLM self-judge)")
- **paper** — the authoritative citation (e.g., "Corbett & Anderson 1995 (BKT); arXiv:2603.02830")

`NODE_META` maps every LangGraph node name (`goal_elicit`, `teach_kp`, `assess_next`, `grade_kp`,
`reteach_kp`, `advance_kp`, `handle_lost`, `diagnose`, `replan`, `select_next`, `induce`,
`induce_scaffold`, plus the full legacy path) to its provenance dict. The template renders these
as inline `<span class="chip">` tags alongside the step's turn content.

## Live mastery and confidence scores

The glass-box shows per-concept mastery (BKT, 0–1) and confidence (number of observations) as
a horizontal bar for each concept in the current session's `learner_state`. These are read live
from the per-session SQLite DB via `litnav/ui/trace.py::build_trace`.

Mastery is surfaced as an **estimate**, consistent with the honest-framing principle — the UI
never claims durable learning, only current BKT probability.

---

## Concept-map SVG

The glass-box renders the concept graph for the current session as an inline SVG, generated by
`litnav/ui/graph_svg.py`. Edge types are visually distinct:

- **Solid edges** — prerequisite relations
- **Dashed edges** — similarity relations (KnowLP fallback)
- **Dotted edges** — induced (off-graph induction)

Concepts are colored by mastery level (heatmap: grey → yellow → green).

---

## Cited evidence per turn

Each teaching turn in the glass-box shows the evidence chunks cited by `teach_kp`. The
`chunk_id` references resolve to real `paper_chunks` rows (guaranteed by OW-5.1); the UI displays
the chunk text and the source paper title.

---

## Cost meter

The glass-box renders a live per-session cost breakdown sourced from `storage/cost_repo.py`
(reading `cost_ledger`). Displayed: cumulative USD, current tier (cheap/frontier), cache hit
rate, and a warning badge when spend ≥ 80% of the session budget cap.

---

## Recommend-next card

At session end (all target concepts done), the `/tutor/{sid}` page appends a **recommend-next
card**: a ranked list of `next_concepts` with prereq status (all-met → "ready now" badge; some
missing → "needs X first") and the `reason` text from the ranker. Sourced from
`litnav/recommend/recommend_next.py` output stored in the session trace.

---

## SSE streaming

The session page polls the `/sessions/{sid}/trace` JSON endpoint via a lightweight SSE-style
`EventSource` (or a polling interval fallback). New steps from the LangGraph graph are appended
incrementally to the chat pane as they complete; the glass-box state refreshes in parallel.

---

## Offline-deterministic demo

The UI runs fully offline (no LLM key needed) against the curated fixture data from
`data/seed/agents_expanded.json` (or `agents_m3.json`). The fixture is seeded into each new
session's per-session SQLite DB at session start:

```python
ag = AgentSession(conn, ckpt, sid, fixture_data())
```

The demo fixture covers `LLM-based autonomous agents` with 5 representative papers, 8+ concepts,
and edges — enough to show the full teach/assess/artifact/recommend-next loop at $0.
